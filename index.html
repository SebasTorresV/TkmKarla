Guía Completa: Despliegue y Captura de Imágenes con WDS

Configurar un servidor WDS en VMware para desplegar y capturar imágenes personalizadas de Windows de manera eficiente y automatizada.

Tiempo estimado: 4-6 horas
7 fases detalladas
VMware + Windows Server

Fase 1: Configuración del Servidor WDS

Instalación y configuración inicial del rol WDS en GRE1-SRV01

1. Instalación del Rol WDS en GRE1-SRV01

Abrimos el Administrador del servidor.

Iniciamos el asistente de "Agregar roles y características".

Seleccionamos el servidor local (GRE1-SRV01) como destino de la instalación.

En la sección "Roles de servidor", marcamos:

Servicios de implementación de Windows

Aceptamos la adición de características necesarias haciendo clic en "Agregar características".

En la sección "Servicios de rol" para WDS, confirmamos que ambos estén marcados:

Servidor de implementación
Servidor de transporte

2. Abrir la Consola de WDS y Agregar el Servidor

Una vez instalado el rol, vamos a "Herramientas" en el Administrador del servidor y seleccionamos "Servicios de implementación de Windows".

Si aparece el mensaje "Este complemento no tiene ningún servidor agregado", hacemos clic derecho en "Servidores" y seleccionamos "Agregar servidor", eligiendo GRE1-SRV01.

3. Configuración Inicial de WDS (Modo Servidor Independiente)

Hacemos clic derecho en GRE1-SRV01 y seleccionamos "Configurar servidor".

🔑 Punto Clave:

En esta fase, seleccionamos el tipo de instalación del servidor WDS. Elegimos:

✅ Servidor independiente

(Esto es importante si no tenemos un controlador de dominio o no deseamos integrar WDS con AD en esta etapa).

Especificamos la ruta para la carpeta RemoteInstall: C:\RemoteInstall

En las "Opciones de configuración de DHCP", marcamos las opciones que WDS sugiera para la integración.

En la "Configuración de respuesta del servidor PXE", seleccionamos:

Responder a todos los equipos cliente (conocidos y desconocidos)

🟢 Estado Actual del Servidor WDS:

  • Rol WDS instalado correctamente
  • Servidor configurado en modo independiente
  • Carpetas básicas de WDS creadas

Fase 2: Preparación y Adición de Imágenes Base

Configuración de imágenes boot.wim e install.wim en WDS

1. Preparación de Archivos WIM de Windows 10

Obtenemos la ISO de Windows 10 (la misma versión que usaremos para la VM de captura).

Montamos la ISO en nuestro equipo host (clic derecho > "Montar").

Navegamos a [Letra de unidad ISO]:\sources\ y localizamos:

boot.wim
install.wim

Copiamos ambos archivos a una ubicación accesible desde el servidor WDS:

C:\ISOS\WN10\boot.wim
C:\ISOS\WN10\install.wim

2. Agregar Imagen de Arranque (boot.wim) a WDS

Abrimos la consola de "Servicios de implementación de Windows".

Clic derecho en "Imágenes de arranque" > "Agregar imagen de arranque..."

Especificamos la ruta: C:\ISOS\WN10\boot.wim

En "Metadatos de imagen", configuramos:

Nombre: WIN10 BOOT

Descripción: Entorno de instalación para Windows 10 desde ISO oficial

Resultado: La imagen WIN10 BOOT aparece en "Imágenes de arranque"

3. Agregar Imagen de Instalación (install.wim) a WDS

Clic derecho en "Imágenes de instalación" > "Agregar grupo de imágenes"

Nombre del grupo: Windows 10

Clic derecho en el grupo "Windows 10" > "Agregar imagen de instalación..."

Especificamos la ruta: C:\ISOS\WN10\install.wim

Seleccionamos únicamente Windows 10 Pro de las ediciones disponibles.

Resultado: Windows 10 Pro aparece en el grupo Windows 10

🟢 Estado Actual del Servidor WDS:

  • Imagen de arranque (WIN10 BOOT) disponible
  • Imagen de instalación (Windows 10 Pro) lista
  • WDS funcional y listo para peticiones PXE

Fase 3: Preparación del Cliente para Captura

Creación y configuración de la VM cliente en VMware

1. Creación de la Máquina Virtual Cliente para Captura (Cliente-Captura-W10)

Abrimos VMware Workstation.

Iniciamos el asistente para crear una nueva máquina virtual (File > New Virtual Machine).

Seleccionamos Custom (advanced) y hacemos clic en Next.

En Hardware compatibility, mantenemos la opción por defecto (ej., "Workstation 17.x").

En Installer disk image file, seleccionamos "I will install the operating system later" (vamos a instalar Windows 10 vía WDS/PXE).

En Guest Operating System, seleccionamos Microsoft Windows y Windows 10 x64.

Nombramos la VM: Cliente-Captura-W10 y elegimos ubicación: D:\Redes1\Captura Cliente WDS

En Processors, asignamos 1 CPU y 2 núcleos por CPU.

En Memory, asignamos 2048 MB (2 GB).

🔑 Punto Clave: Configuración de Red

Aquí es donde aseguramos la comunicación con WDS:

Seleccionamos "Use network connection: Bridged (Automatic)".

"Bridged" es el modo más confiable para PXE ya que coloca la VM directamente en tu red física, permitiendo acceso al DHCP y WDS.

En I/O Controller Types, seleccionamos LSI Logic SAS (Recommended).

En Disk Types, seleccionamos Create a new virtual disk.

En Disk Size, especificamos 30 GB y seleccionamos "Store virtual disk as a single file".

En la pantalla de resumen, hacemos clic en Customize Hardware... para verificar:

  • Network Adapter en modo Bridged
  • "Connect at power on" activado
  • (Opcional) Configurar arranque por red en Boot Options

2. Instalación y Preparación de Windows 10 en Cliente-Captura-W10

Instalar Windows 10: Encendemos la VM Cliente-Captura-W10. Durante el arranque, forzamos el arranque PXE (F12) para conectar a WDS (GRE1-SRV01). Seleccionamos WIN10 BOOT y luego Windows 10 Pro.

Personalización: Una vez instalado Windows 10 Pro:

  • Instalamos VMware Tools
  • Actualizamos con Windows Update
  • Instalamos software base (navegadores, Office, etc.)
  • Configuramos sistema (fondo, idioma, energía, etc.)
  • Limpieza de disco (cleanmgr.exe)

📸 Tomar una Instantánea (Snapshot): ¡CRÍTICO!

Antes de ejecutar Sysprep, creamos una instantánea de la VM (VM > Snapshot > Take Snapshot...) con nombre "W10 Personalizado Antes de Sysprep". Esto nos permite revertir si algo sale mal.

3. Ejecutar Sysprep en Cliente-Captura-W10

Dentro de la VM, abrimos el Símbolo del sistema (CMD) como Administrador.

Navegamos a la carpeta de Sysprep: cd %windir%\system32\sysprep

Ejecutamos Sysprep con los parámetros correctos:

sysprep.exe /oobe /generalize /shutdown

/oobe: Prepara para la Experiencia Rápida (OOBE)

/generalize: Elimina información específica del sistema (SID, nombres, controladores)

/shutdown: Apaga la máquina inmediatamente después

⚠️ ¡Advertencia!

Una vez que la VM se apaga después de Sysprep, NO la inicies de nuevo en su sistema operativo. Debe permanecer apagada hasta el momento de la captura PXE.

Fase 4: Captura de la Imagen Personalizada

Proceso de captura con WDS y configuración del asistente

1. Agregar Imagen de Captura a WDS en GRE1-SRV01

Abrimos la consola de "Servicios de implementación de Windows" en GRE1-SRV01.

Vamos a "Imágenes de arranque".

Clic derecho en WIN10 BOOT > "Crear imagen de captura..."

En el asistente:

  • Ruta: C:\Temp\W10_Captura.wim
  • Nombre: "W10 Captura - WDS"
  • Descripción apropiada

Clic derecho en "Imágenes de arranque" > "Agregar imagen de arranque..."

Seleccionamos el archivo de captura: C:\Temp\W10_Captura.wim

Resultado: Ahora tenemos WIN10 BOOT y W10 Captura - WDS en "Imágenes de arranque"

2. Iniciar el Cliente y Forzar Arranque PXE

Con la VM Cliente-Captura-W10 apagada, en VMware Workstation:

  • Verificar Network Adapter en "Bridged"
  • Marcar "Force BIOS setup" en Boot Options

Encendemos la VM Cliente-Captura-W10.

En el BIOS/UEFI, movemos "Network Boot" / "PXE Boot" a la primera posición. Guardamos (F10).

🔑 ¡CRÍTICO!

Cuando aparezca "Press F12 for network service boot", presionamos F12 rápidamente para forzar el arranque PXE.

3. Seleccionar la Imagen de Captura y Autenticarse

En el "Windows Boot Manager", seleccionamos W10 Captura - WDS (¡paso crucial!).

Presionamos ENTER.

Windows PE se carga e inicia el "Asistente para captura de imágenes de WDS".

En "Escribir credenciales de red":

Usuario: MIDOMINIO\Administrador (o administrador@midominio.com)

Contraseña: La contraseña correspondiente

Si la autenticación es exitosa, avanza a "Ubicación de imagen nueva".

4. Configurar la Ubicación de la Imagen Nueva

En "Ubicación de imagen nueva":

  • Marcar ✅ "Cargar imagen en un servidor de WDS (opcional)"
  • Servidor: GRE1-SRV01 (o IP: 192.168.197.11)
  • Clic en "Conectar"
  • Grupo: "Windows 10"

✅ Estado Actual:

El asistente de captura está listo para iniciar la grabación de la imagen de Cliente-Captura-W10 y subirla a WDS (GRE1-SRV01) dentro del grupo "Windows 10".

Siguiente Paso: Proceso de Captura

El asistente comenzará el proceso de grabación. Verás una barra de progreso. Esto puede tardar un tiempo considerable.

Una vez finalizado, el cliente se apagará automáticamente.

Verificar en WDS (GRE1-SRV01) > "Imágenes de instalación" > "Windows 10" que la nueva imagen personalizada aparece con estado "Disponible".

Fase 5: Creación del Archivo unattend.xml

Automatización de instalaciones con Windows ADK y WSIM

1. Preparación de Herramientas - Windows ADK

En GRE1-SRV01, abrimos el navegador y buscamos: "Descargar Windows ADK para Windows 10".

Accedemos al enlace oficial de Microsoft Learn y descargamos el instalador del ADK.

Ejecutamos el instalador (adksetup.exe) y seguimos el asistente.

En "Seleccione las características":

✅ Herramientas de implementación (Deployment Tools)

Este componente es indispensable para WSIM

2. Abrir Windows System Image Manager (WSIM)

Vamos al menú Inicio y buscamos la carpeta "Windows Kits".

Hacemos clic en "Windows System Image Manager".

Verificamos que la ventana muestre "Windows System Image Manager" en su título.

3. Cargar la Imagen de Windows (install.wim)

En WSIM: Archivo > Seleccionar imagen de Windows...

Navegamos a: C:\ISOS\WN10\install.wim

Seleccionamos el archivo y hacemos clic en "Abrir".

Si contiene múltiples ediciones, seleccionamos Windows 10 Pro.

📁 Creación de Archivo de Catálogo

WSIM mostrará: "No se encuentra el archivo de catálogo... ¿Desea crear un archivo de catálogo?"

✅ Hacemos clic en "Sí"

• Si hay errores de permisos, asegurar "Control total" sobre C:\ISOS\WN10\

• La generación puede tardar 10-15 minutos

Fase 6: Configuración del Archivo de Respuestas (unattend.xml)

Configuración detallada de todos los pases de instalación

1. Crear Nuevo Archivo de Respuestas

En WSIM: Archivo > Nuevo archivo de respuestas

El panel central se activa mostrando los "Pases" de configuración (windowsPE, specialize, oobeSystem, etc.)

2. Configurar Pase 1 windowsPE

2.1 Idioma y Regionalización

• Expandir Components > amd64_Microsoft-Windows-International-Core-WinPE

• Clic derecho > "Agregar configuración al pase 1 windowsPE"

• Configurar en Propiedades:

InputLocale: es-ES

SystemLocale: es-ES

UILanguage: es-ES

UILanguageFallback: es-ES

UserLocale: es-ES

2.2 Aceptar EULA y Clave de Producto

• Components > amd64_Microsoft-Windows-Setup > UserData

• Clic derecho > "Agregar configuración al pase 1 windowsPE"

• Configurar en Propiedades:

AcceptEula: True (Crucial para automatización)

ProductKey: (Opcional - dejar en blanco si no tienes clave)

2.3 Configuración de Disco (Crítico)

Paso A: Agregar DiskConfiguration

• Components > amd64_Microsoft-Windows-Setup > DiskConfiguration

• Clic derecho > "Agregar configuración al pase 1 windowsPE"

Paso B: Configurar Disco a Borrar

• Clic derecho en DiskConfiguration > "Insertar nuevo disco"

• DiskID: 0, WillWipeDisk: True

Paso C: Crear 4 Particiones (CreatePartitions)

• Insertar 4 CreatePartition en el Disk

• Partición 1: Order=1, Size=500, Type=Primary

• Partición 2: Order=2, Size=100, Type=EFI

• Partición 3: Order=3, Size=128, Type=MSR

• Partición 4: Order=4, Extend=True, Type=Primary

Paso D: Formatear Particiones (ModifyPartitions)

• Insertar 4 ModifyPartition en el Disk

• ModifyPartition 1: Order=1, Format=NTFS, Label=Recovery, PartitionID=1

• ModifyPartition 2: Order=2, Format=FAT32, Label=System, PartitionID=2

• ModifyPartition 3: Order=3, PartitionID=3 (sin Format)

• ModifyPartition 4: Order=4, Format=NTFS, Label=Windows, PartitionID=4, Letter=C

2.4 Destino de Instalación (InstallTo)

• Components > amd64_Microsoft-Windows-Setup > ImageInstall > OSImage > InstallTo

• Clic derecho > "Agregar configuración al pase 1 windowsPE"

• Configurar:

DiskID: 0

PartitionID: 4 (Partición del sistema operativo)

3. Configurar Pase 3 specialize

Nombre de Equipo y Zona Horaria

• Components > amd64_Microsoft-Windows-Shell-Setup

• Clic derecho > "Agregar configuración al pase 3 specialize"

• Configurar en Propiedades:

ComputerName: PC%RAND% (nombre aleatorio)

TimeZone: Central America Standard Time

4. Configurar Pase 7 oobeSystem

4.1 Omitir OOBE

• Components > amd64_Microsoft-Windows-Shell-Setup > OOBE

• Clic derecho > "Agregar configuración al pase 7 oobeSystem"

• Configurar:

HideEULAPage: True

ProtectYourPC: 3

SkipMachineOOBE: True

SkipUserOOBE: True

4.2 Crear Cuenta de Administrador Local

• Components > amd64_Microsoft-Windows-Shell-Setup > UserAccounts > LocalAccounts

• Clic derecho > "Agregar configuración al pase 7 oobeSystem"

• Clic derecho en LocalAccounts > "Insertar nueva LocalAccount"

• Configurar la LocalAccount:

Name: Administrador

DisplayName: Administrador Local

Group: Administrators

• Seleccionar Password bajo LocalAccount y configurar:

Value: TuContraseñaSegura123

4.3 Inicio de Sesión Automático (Opcional)

• Components > amd64_Microsoft-Windows-Shell-Setup > AutoLogon

• Clic derecho > "Agregar configuración al pase 7 oobeSystem"

• Configurar:

Enabled: True

LogonCount: 1

Username: Administrador

• Seleccionar Password bajo AutoLogon:

Value: TuContraseñaSegura123 (la misma contraseña)

Fase 7: Validación e Integración del unattend.xml

Validación final e integración con WDS

1. Validar el Archivo de Respuestas

En WSIM: Herramientas > Validar archivo de respuestas

Revisar el panel "Mensajes" en la parte inferior.

✅ Resultado Esperado:

"No hay errores ni advertencias en el archivo de respuestas"

Advertencias (ícono amarillo) son aceptables, pero NO errores (ícono rojo)

Si hay errores, WSIM indicará la descripción y ubicación para corregirlos.

2. Guardar el Archivo de Respuestas

Una vez validado correctamente: Archivo > Guardar archivo de respuestas como...

Configurar el guardado:

Nombre: unattend.xml

Ubicación: C:\RemoteInstall\Images\Windows 10\

Hacer clic en "Guardar".

3. Integrar con WDS

Abrir la consola "Servicios de implementación de Windows" en GRE1-SRV01.

Navegar a: Servidores > GRE1-SRV01 > Imágenes de instalación > Windows 10

Hacer clic derecho en Windows 10 Pro > "Propiedades".

En la pestaña "General":

  • Marcar ✅ "Permitir la instalación de la imagen en modo desatendido"
  • Hacer clic en "Examinar..."
  • Seleccionar: C:\RemoteInstall\Images\Windows 10\unattend.xml
  • Hacer clic en "Abrir"

Verificar que la ruta del unattend.xml aparezca en el campo.

Hacer clic en "Aplicar" y luego "Aceptar".

🎉 ¡Integración Completada!

El archivo unattend.xml está ahora vinculado a la imagen Windows 10 Pro en WDS. Las futuras instalaciones serán completamente automatizadas.

📄 Explicación Completa del Archivo unattend.xml

Análisis detallado de cada sección y configuración del archivo de respuestas

Este archivo permite realizar una instalación completamente desatendida de Windows 10 a través de WDS (Windows Deployment Services). Está dividido por secciones llamadas "passes", que representan diferentes fases de la instalación.

🧩 1. <settings pass="windowsPE">

Esta es la primera fase que se ejecuta cuando el equipo cliente arranca desde red (PXE) usando WDS.

🗺️ Configuración Regional e Idioma

<Microsoft-Windows-International-Core-WinPE> <InputLocale>es-ES</InputLocale> <SystemLocale>es-ES</SystemLocale> <UILanguage>es-ES</UILanguage> <UILanguageFallback>es-ES</UILanguageFallback> <UserLocale>es-ES</UserLocale> </Microsoft-Windows-International-Core-WinPE>

Define idioma, región y distribución del teclado: Español (España).

💾 Configuración de Particiones de Disco

<DiskConfiguration> <DiskID>0</DiskID> <WillWipeDisk>true</WillWipeDisk>

Indica que se usará el disco físico 0 y se borrará completamente (WillWipeDisk).

Particiones que se crean:
  • Recovery (450 MB) – para recuperación del sistema
  • EFI (100 MB, FAT32) – necesaria si se usa firmware UEFI
  • MSR (128 MB) – partición reservada por Microsoft
  • Windows (resto del disco) – donde se instalará Windows
<ModifyPartition ... Format="NTFS" Label="Windows" Letter="C" />

• Partición 4 (Windows) se formatea como NTFS, con letra C

• Partición 2 (EFI) es FAT32, necesaria para el arranque

• La partición Recovery se marca con un TypeID especial que identifica su función

📦 Instalación de la Imagen de Windows

<ImageInstall> <OSImage> <InstallTo> <DiskID>0</DiskID> <PartitionID>4</PartitionID> </InstallTo> </OSImage> </ImageInstall>

Especifica que Windows se instalará en la partición 4 del disco 0.

🔐 Conexión al Servidor WDS

<WindowsDeploymentServices> <Login> <Credentials> <Domain>gre1.local</Domain> <Username>Administrador</Username> <Password>Matias1234</Password> </Credentials> </Login>

• Usa las credenciales del dominio (gre1.local) para conectarse al servidor WDS

• El usuario Administrador es una cuenta del dominio usada para autenticar y descargar la imagen

Selección de imagen:
<InstallImage> <Filename>install.wim</Filename> <ImageGroup>WN10 Pro</ImageGroup> <ImageName>Windows 10 Pro</ImageName> </InstallImage>

Se selecciona la imagen Windows 10 Pro desde el archivo install.wim dentro del grupo WN10 Pro.

🧩 2. <settings pass="specialize">

Esta fase ocurre justo después de que Windows ha copiado los archivos e iniciado por primera vez.

🖥️ Nombre del Equipo y Zona Horaria

<ComputerName>*</ComputerName> <TimeZone>Central America Standard Time</TimeZone>

• El nombre del equipo se genera automáticamente (*)

• Se establece la zona horaria de Centroamérica (UTC-6), útil para El Salvador

🧩 3. <settings pass="oobeSystem">

Esta fase es la Out-Of-Box Experience (OOBE), es decir, lo que normalmente verías al encender por primera vez un equipo nuevo. Aquí se evita todo eso.

🔐 Autologon (inicio de sesión automático)

<AutoLogon> <Username>Sebas</Username> <Password> <Value>Matias1234</Value> <PlainText>true</PlainText> </Password> </AutoLogon>

Windows iniciará sesión automáticamente con la cuenta Sebas y la contraseña Matias1234.

👤 Cuenta Local Creada

<LocalAccount wcm:action="add"> <Name>Sebas</Name> <Group>Administrators</Group> <DisplayName>Sebas</DisplayName> <Password> <Value>Matias1234</Value> <PlainText>true</PlainText> </Password> </LocalAccount>

• Se crea una cuenta local llamada Sebas, con privilegios de administrador

• Esta cuenta se usa para trabajar en el equipo después de la instalación

• Es la misma cuenta configurada para el autologon

📵 Configuraciones que Ocultan Pantallas en OOBE

<HideEULAPage>true</HideEULAPage> <HideLocalAccountScreen>true</HideLocalAccountScreen> <HideOEMRegistrationScreen>true</HideOEMRegistrationScreen> <HideOnlineAccountScreens>true</HideOnlineAccountScreens> <HideWirelessSetupInOOBE>true</HideWirelessSetupInOOBE>
Evita que se muestren las siguientes pantallas:
  • Contrato de licencia (EULA)
  • Creación de cuenta local
  • Registro del fabricante
  • Inicio de sesión con cuenta Microsoft
  • Configuración de redes inalámbricas
<NetworkLocation>Work</NetworkLocation>

Establece que el perfil de red será "Trabajo", lo que permite recursos compartidos sin restricciones como en redes domésticas.

🧩 4. <cpi:offlineImage>

<cpi:offlineImage cpi:source="wim:c:/isos/wn10/install.wim#Windows 10 Pro" />

Es una línea interna usada por el Windows System Image Manager (WSIM) para saber con qué imagen .wim se está trabajando.

✅ Resultado Final

Con este archivo, cuando una máquina cliente arranca desde red (PXE) y se conecta al servidor WDS:

Se borrará el disco y se crearán las particiones necesarias

Se instalará la imagen de Windows 10 Pro sin intervención

Se configurará el idioma y zona horaria automáticamente

Se creará la cuenta Sebas con privilegios de administrador

Se omitirá toda la configuración de OOBE

El sistema iniciará sesión automáticamente con el usuario Sebas

🎉 ¡Proceso Completado!

Has configurado exitosamente un servidor WDS y capturado una imagen personalizada de Windows 10. Tu infraestructura está lista para desplegar esta imagen en múltiples clientes de manera eficiente.
